# CONTENT_RULES.md - Reglas de Generación de Contenido para Tramitia

Este documento define las reglas estrictas para la generación de contenido de blog en tramitia.es. Estas reglas son OBLIGATORIAS y deben seguirse al pie de la letra.

> **IMPORTANTE:** Todo el contenido DEBE escribirse con ortografía española correcta, incluyendo tildes (á, é, í, ó, ú), eñes (ñ) y signos de interrogación/exclamación de apertura (¿, ¡).

---

## 1. ESTRUCTURA OBLIGATORIA DEL ARTÍCULO

Todo artículo DEBE seguir esta estructura exacta:

```
1. PARRAFO INTRODUCTORIO (Respuesta directa - 2-3 oraciones)
   - Responde la pregunta principal del titulo en las primeras 50 palabras
   - Incluye la keyword principal
   - Optimizado para featured snippets y respuestas de IA

2. TABLA DE CONTENIDOS (opcional, para articulos >1500 palabras)
   - Lista de H2 como enlaces ancla
   - Solo si el articulo tiene 4+ secciones H2

3. CUERPO DEL ARTICULO (secciones H2)
   - Minimo 3 secciones H2
   - Maximo 7 secciones H2
   - Cada H2 puede tener 0-3 subsecciones H3

4. SECCION DE PROCESO/PASOS (si aplica)
   - Formato de lista numerada
   - Cada paso con H3 si es extenso
   - Incluir tiempos estimados cuando sea posible

5. SECCION FAQ (OBLIGATORIA)
   - Minimo 4 preguntas
   - Maximo 8 preguntas
   - Formato pregunta-respuesta

6. SECCION CTA FINAL
   - Siempre antes de las FAQs o al final
   - Enlace a formulario de contacto
   - Mencion de WhatsApp como alternativa

7. ARTICULOS RELACIONADOS (automatico via layout)
```

### Longitud segun tipo de contenido:

| Tipo de articulo | Palabras | Tiempo lectura |
|------------------|----------|----------------|
| Guia completa | 2000-3000 | 10-15 min |
| Guia de proceso | 1500-2000 | 7-10 min |
| Articulo informativo | 1000-1500 | 5-7 min |
| Noticia/Actualizacion | 500-800 | 3-4 min |

## 2. REGLAS DE TITULO Y META DESCRIPTION

### Titulo (title tag y H1)

**OBLIGATORIO:**
- Longitud: 50-60 caracteres (maximo 60)
- Keyword principal en las primeras 3 palabras
- Incluir pais o tipo de documento cuando sea especifico
- Evitar palabras vacias al inicio (El, La, Los, Las, Un, Una)

**FORMATOS PERMITIDOS:**

```
# Para guias de proceso:
"Como [accion] [documento] de [pais] en [anio]"
"[Documento] de [pais]: Guia completa [anio]"
"Apostillar [documento] [pais] desde Espana: Guia [anio]"

# Para informativos:
"[Tema]: Todo lo que necesitas saber en [anio]"
"[Numero] requisitos para [accion] [documento]"
"[Documento] para [tramite]: Requisitos y proceso"

# Para comparativos:
"[Opcion A] vs [Opcion B]: Diferencias clave"
"Tipos de [documento]: Cual necesitas para [tramite]"
```

**EJEMPLOS CORRECTOS:**
- "Apostillar partida de nacimiento Venezuela: Guia 2025"
- "Antecedentes penales Argentina para nacionalidad espanola"
- "Como apostillar titulo universitario colombiano en 2025"

**EJEMPLOS INCORRECTOS:**
- "La guia definitiva para apostillar documentos" (muy generico)
- "Todo sobre apostillas" (sin especificidad)
- "Apostilla Venezuela" (muy corto, sin contexto)

### Meta Description

**OBLIGATORIO:**
- Longitud: 145-155 caracteres (maximo 160)
- Keyword principal en las primeras 20 palabras
- Incluir beneficio o propuesta de valor
- Terminar con CTA implicito o explicito
- NO usar mayusculas innecesarias
- NO usar signos de exclamacion

**FORMATO:**
```
[Keyword principal] + [Que resuelve] + [Beneficio/Diferenciador] + [CTA implicito]
```

**EJEMPLO CORRECTO:**
```
Apostillar partida de nacimiento de Venezuela desde Espana sin viajar.
Proceso 100% online, equipo local en Caracas. Consulta gratis.
```

**EJEMPLO INCORRECTO:**
```
APOSTILLA TU DOCUMENTO HOY! Somos los mejores en apostillas.
Contactanos ya para mas informacion sobre nuestros servicios.
```

## 3. REGLAS DE HEADINGS (H1-H3)

### H1 (Titulo principal)
- SOLO UNO por pagina (automatico desde el titulo)
- Identico o muy similar al title tag
- Keyword principal obligatoria

### H2 (Secciones principales)

**CANTIDAD:** 3-7 por articulo

**REGLAS:**
- Keyword principal o secundaria en al menos 2 de los H2
- Usar preguntas en 1-2 H2 (mejora CTR en SERPs)
- Maximo 70 caracteres
- No repetir las mismas palabras en multiples H2

**H2 OBLIGATORIOS segun tipo de contenido:**

Para GUIAS DE PROCESO:
```
- "Que es [documento/proceso]" o "Que necesitas saber sobre..."
- "Requisitos para [accion]"
- "Paso a paso: Como [accion]"
- "Cuanto cuesta [proceso]" o "Precios y tiempos"
- "Preguntas frecuentes sobre [tema]"
```

Para ARTICULOS INFORMATIVOS:
```
- Definicion o contexto del tema
- Aplicacion practica para el usuario
- Casos de uso especificos
- "Preguntas frecuentes sobre [tema]"
```

### H3 (Subsecciones)

**CANTIDAD:** 0-3 por cada H2

**REGLAS:**
- Usar para desglosar pasos o listas complejas
- Incluir keywords long-tail cuando sea natural
- Maximo 60 caracteres
- No usar H3 si el contenido bajo el H2 es menor a 150 palabras

## 4. REGLAS DE CONTENIDO

### Tono y voz

**OBLIGATORIO:**
- Tono: Profesional pero cercano
- Voz: Segunda persona (tu, te, tus) - NUNCA usted
- Registro: Formal-neutro (espanol de Espana, entendible para latinoamericanos)
- Perspectiva: Experto que guia, no vendedor agresivo

**EJEMPLOS DE TONO:**

CORRECTO:
```
"Si vives en Espana y necesitas apostillar tu partida de nacimiento
venezolana, no tienes que volver a tu pais. Te explicamos como
gestionarlo desde aqui."
```

INCORRECTO:
```
"Estimado usuario, en Tramitia le ofrecemos los mejores servicios
de apostilla del mercado. No dude en contactarnos."
```

### Estructura de parrafos

**REGLAS:**
- Longitud: 2-4 oraciones por parrafo (maximo 100 palabras)
- Una idea principal por parrafo
- Usar parrafos cortos para informacion clave
- Espacio entre parrafos para facilitar escaneo

### Listas y tablas

**USAR LISTAS CUANDO:**
- Enumerar requisitos (minimo 3 items)
- Listar documentos necesarios
- Desglosar pasos de un proceso
- Comparar opciones

**FORMATO DE LISTAS:**
```markdown
**Documentos necesarios:**
- Partida de nacimiento original
- Copia del pasaporte vigente
- Formulario de solicitud completado
- Comprobante de pago de tasas
```

**USAR TABLAS CUANDO:**
- Comparar precios o tiempos entre opciones
- Mostrar requisitos por pais/tipo de documento
- Presentar plazos y costos

**FORMATO DE TABLAS:**
```markdown
| Documento | Tiempo estimado | Costo aproximado |
|-----------|-----------------|------------------|
| Partida de nacimiento | 15-20 dias | 80-120 EUR |
| Antecedentes penales | 20-30 dias | 100-150 EUR |
```

### Uso de negritas y enfasis

**USAR NEGRITAS PARA:**
- Terminos clave que el usuario busca
- Numeros importantes (precios, plazos)
- Nombres de instituciones
- Advertencias o datos criticos

**NO USAR NEGRITAS PARA:**
- Parrafos completos
- Mas de 3-4 elementos por seccion
- Enfatizar CTAs (el boton ya destaca)

## 5. REGLAS DE FAQ

### Cantidad
- Minimo: 4 preguntas
- Optimo: 5-6 preguntas
- Maximo: 8 preguntas

### Formato de preguntas

**OBLIGATORIO:**
- Comenzar con palabra interrogativa (Que, Como, Cuando, Donde, Cuanto, Por que, Cual)
- Incluir keyword o entidad relevante
- Maximo 80 caracteres por pregunta
- Usar lenguaje natural (como lo buscaria el usuario)

**PREGUNTAS OBLIGATORIAS segun tipo de contenido:**

Para articulos sobre APOSTILLA DE DOCUMENTOS:
```
1. Cuanto cuesta apostillar [documento] de [pais]?
2. Cuanto tiempo tarda la apostilla de [documento]?
3. Puedo apostillar [documento] sin volver a [pais]?
4. Que requisitos necesito para apostillar [documento]?
5. [Pregunta especifica del tema]
```

Para articulos sobre TRAMITES EN ESPANA:
```
1. Que documentos necesito para [tramite]?
2. Donde se realiza [tramite] en Espana?
3. Cuanto cuesta [tramite]?
4. Cuanto tiempo tarda [tramite]?
5. [Pregunta especifica del tema]
```

### Formato de respuestas

**OBLIGATORIO:**
- Primera oracion: Respuesta directa (Si/No o dato concreto)
- Extension: 40-80 palabras por respuesta
- Incluir dato numerico cuando sea posible (precio, tiempo)
- No usar listas dentro de respuestas FAQ
- Terminar con informacion util adicional, no con CTA

**EJEMPLO CORRECTO:**
```
Pregunta: Cuanto cuesta apostillar una partida de nacimiento de Venezuela?
Respuesta: El costo total oscila entre 80 y 120 euros, dependiendo
del tipo de servicio (regular o express). Este precio incluye la
gestion ante el Ministerio de Relaciones Exteriores en Caracas,
la apostilla de La Haya y el envio del documento a Espana.
```

**EJEMPLO INCORRECTO:**
```
Pregunta: Cuanto cuesta?
Respuesta: Depende de varios factores. Contactanos para un presupuesto
personalizado. Tenemos los mejores precios del mercado.
```

## 6. REGLAS DE ENLACES INTERNOS Y CTAs

### Enlaces internos

**CANTIDAD POR ARTICULO:**
- Minimo: 2 enlaces internos
- Optimo: 3-5 enlaces internos
- Maximo: 7 enlaces internos

**DESTINOS PRIORITARIOS:**
1. Landing page del pais relacionado (`/apostilla-venezuela`, etc.)
2. Articulos de blog relacionados por tema o pais
3. Homepage (solo si es relevante contextualmente)

**UBICACION:**
- Al menos 1 en el primer tercio del articulo
- Al menos 1 en la seccion de proceso/pasos
- Nunca en la seccion FAQ (rompe el schema)

**ANCHOR TEXT:**
- Usar texto descriptivo, no "clic aqui" o "leer mas"
- Incluir keyword del articulo destino cuando sea natural
- Variar los anchor texts (no repetir el mismo)

**EJEMPLOS:**
```markdown
CORRECTO:
"Si necesitas apostillar tu [partida de nacimiento de Venezuela](/apostilla-venezuela),
el proceso es similar."

INCORRECTO:
"Para mas informacion, [haz clic aqui](/apostilla-venezuela)."
```

### CTAs (Calls to Action)

**CANTIDAD:**
- 1 CTA principal al final del articulo (antes de FAQ)
- 1 CTA secundario si el articulo supera las 1500 palabras (mitad del articulo)

**FORMATO DEL CTA PRINCIPAL:**
```markdown
---

## Apostilla tus documentos sin complicaciones

En Tramitia nos encargamos de todo el proceso de apostilla desde Espana.
Nuestro equipo en [pais] gestiona tu [documento] sin que tengas que viajar.

[Solicita tu presupuesto gratis](/#contacto) o escribenos por
[WhatsApp](https://wa.me/34XXXXXXXXX) para resolver tus dudas.

---
```

**REGLAS DEL CTA:**
- Usar H2 para el titulo del CTA
- Maximo 3 oraciones
- Incluir 2 opciones de contacto (formulario + WhatsApp)
- NO usar lenguaje urgente o de escasez ("solo hoy", "ultimas plazas")
- NO usar mayusculas o signos de exclamacion

## 7. REGLAS DE SCHEMA.ORG Y DATOS ESTRUCTURADOS

### Schema obligatorio para cada articulo

Todo post de blog DEBE incluir estos schemas (generados automaticamente por el layout):

```json
// 1. Article Schema
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "[Titulo del articulo]",
  "description": "[Meta description]",
  "datePublished": "[Fecha ISO]",
  "dateModified": "[Fecha actualizacion ISO]",
  "author": {
    "@type": "Organization",
    "name": "Tramitia"
  },
  "publisher": {
    "@type": "Organization",
    "name": "Tramitia",
    "logo": {
      "@type": "ImageObject",
      "url": "https://tramitia.es/logo.svg"
    }
  },
  "mainEntityOfPage": {
    "@type": "WebPage",
    "@id": "https://tramitia.es/blog/[slug]"
  }
}

// 2. BreadcrumbList Schema
{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    {"@type": "ListItem", "position": 1, "name": "Inicio", "item": "https://tramitia.es"},
    {"@type": "ListItem", "position": 2, "name": "Blog", "item": "https://tramitia.es/blog"},
    {"@type": "ListItem", "position": 3, "name": "[Titulo]", "item": "https://tramitia.es/blog/[slug]"}
  ]
}

// 3. FAQPage Schema (SI el articulo tiene seccion FAQ)
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "[Pregunta]",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "[Respuesta]"
      }
    }
  ]
}
```

### Schema adicional segun tipo de contenido

**Para articulos tipo GUIA (HowTo):**
```json
{
  "@context": "https://schema.org",
  "@type": "HowTo",
  "name": "[Titulo del proceso]",
  "description": "[Descripcion breve]",
  "totalTime": "P[X]D",
  "estimatedCost": {
    "@type": "MonetaryAmount",
    "currency": "EUR",
    "value": "[precio]"
  },
  "step": [
    {
      "@type": "HowToStep",
      "position": 1,
      "name": "[Nombre del paso]",
      "text": "[Descripcion del paso]"
    }
  ]
}
```

**Nota:** El schema HowTo debe agregarse manualmente en Sanity o mediante un campo dedicado.

## 8. REGLAS DE AEO (Answer Engine Optimization)

Para aparecer en respuestas de ChatGPT, Perplexity, Google AI Overviews:

### Estructura de respuesta directa

**OBLIGATORIO en el primer parrafo:**
- Responder la pregunta del titulo en las primeras 50 palabras
- Usar formato "definicion + dato clave + contexto"
- Incluir numero o dato verificable cuando sea posible

**EJEMPLO:**
```
Titulo: "Cuanto cuesta apostillar una partida de nacimiento de Venezuela"

Primer parrafo:
"Apostillar una partida de nacimiento de Venezuela cuesta entre 80 y 120
euros cuando se gestiona desde Espana. Este precio incluye la obtencion
del documento en el registro civil, la apostilla ante el Ministerio de
Relaciones Exteriores y el envio certificado a tu direccion en Espana."
```

### Formato de contenido citable

**CREAR BLOQUES "CITABLES":**
```markdown
**Dato clave:** La apostilla de La Haya es valida en los 124 paises
miembros del Convenio de La Haya de 1961, incluyendo Espana y todos
los paises de la Union Europea.
```

### Entidades y datos estructurados

**INCLUIR SIEMPRE:**
- Nombres oficiales de instituciones
- Numeros de leyes o convenios cuando sea relevante
- Fechas de actualizacion
- Fuentes oficiales

**EJEMPLO:**
```
"Segun el Convenio de La Haya de 1961 (Convenio de la Apostilla),
los documentos publicos emitidos en un pais miembro son validos
en todos los demas paises firmantes una vez apostillados."
```

### Actualizacion y vigencia

**OBLIGATORIO:**
- Incluir el ano en el titulo cuando el contenido cambia anualmente
- Agregar fecha de ultima actualizacion visible
- Revisar y actualizar el contenido cada 6-12 meses

## 10. ERRORES A EVITAR (LISTA NEGRA)

### Contenido

**PROHIBIDO:**
- [ ] Usar emojis en cualquier parte del contenido
- [ ] Copiar contenido de otras webs (plagio)
- [ ] Inventar datos, precios o plazos sin verificar
- [ ] Dar consejos legales especificos
- [ ] Garantizar resultados de tramites ("te aprobaran seguro")
- [ ] Hablar negativamente de competidores
- [ ] Usar "nosotros somos los mejores/unicos/primeros"

### SEO

**PROHIBIDO:**
- [ ] Keyword stuffing (repetir keyword mas de 5 veces en 1000 palabras)
- [ ] Titulos clickbait o enganosos
- [ ] Meta descriptions duplicadas entre articulos
- [ ] Contenido duplicado entre articulos
- [ ] Enlaces rotos o a paginas inexistentes
- [ ] Mas de 7 enlaces internos por articulo
- [ ] Enlaces externos a competidores directos

### Formato

**PROHIBIDO:**
- [ ] Parrafos de mas de 5 lineas
- [ ] Secciones sin ningun H2/H3
- [ ] Mas de 500 palabras sin un subheading
- [ ] Imagenes sin alt text
- [ ] Listas con mas de 10 items sin dividir
- [ ] Usar mayusculas para enfatizar (IMPORTANTE, ATENCION)
- [ ] Signos de exclamacion en titulos o CTAs

### Schema/Datos estructurados

**PROHIBIDO:**
- [ ] FAQs sin formato pregunta-respuesta
- [ ] Preguntas FAQ que no empiecen con interrogativo
- [ ] Respuestas FAQ de mas de 300 caracteres
- [ ] HowTo sin pasos numerados
- [ ] Fechas en formato incorrecto (usar ISO 8601)

### UX/Conversion

**PROHIBIDO:**
- [ ] Mas de 2 CTAs por articulo
- [ ] Popups o modales intrusivos
- [ ] Contenido oculto detras de formularios
- [ ] Precios falsos o enganosos
- [ ] Crear urgencia artificial ("solo hoy", "ultimas 3 plazas")

Antes de publicar cualquier articulo, verificar:

### Estructura
- [ ] El articulo tiene entre 1000-3000 palabras
- [ ] Hay un parrafo introductorio con respuesta directa
- [ ] El articulo tiene 3-7 secciones H2
- [ ] Hay una seccion FAQ con 4-8 preguntas
- [ ] Hay al menos 1 CTA claro

### SEO
- [ ] Titulo tiene 50-60 caracteres con keyword principal
- [ ] Meta description tiene 145-155 caracteres
- [ ] Keyword principal aparece en H1, primer parrafo y al menos 2 H2
- [ ] Hay 2-5 enlaces internos
- [ ] Imagen principal tiene alt text descriptivo

### Schema
- [ ] Article schema incluye todos los campos obligatorios
- [ ] Breadcrumb schema esta correcto
- [ ] FAQ schema formateado correctamente (si aplica)
- [ ] Fechas en formato ISO 8601

### AEO
- [ ] Primer parrafo responde la pregunta principal
- [ ] Incluye datos/numeros verificables
- [ ] Menciona entidades oficiales por nombre
- [ ] Contenido actualizado (ano correcto)

### Calidad
- [ ] Sin errores ortograficos o gramaticales
- [ ] Sin emojis
- [ ] Tono profesional y cercano
- [ ] Informacion verificada y actualizada

## CAMPOS DE SANITY A COMPLETAR

Al crear un post en Sanity, estos campos son OBLIGATORIOS:

| Campo | Tipo | Requerido | Notas |
|-------|------|-----------|-------|
| title | string | SI | 50-60 caracteres |
| slug | slug | SI | Generado automaticamente |
| description | text | SI | 145-160 caracteres (meta description) |
| mainImage | image | RECOMENDADO | Con alt text |
| category | string | SI | apostillas, documentos, tramites, guias, noticias |
| country | string | RECOMENDADO | venezuela, argentina, colombia, etc. |
| documentType | string | RECOMENDADO | partida-nacimiento, antecedentes-penales, etc. |
| body | array (Portable Text) | SI | Contenido del articulo |
| keywords | array | SI | 3-6 keywords |
| publishedAt | datetime | SI | Fecha de publicacion |
| draft | boolean | NO | Por defecto: false |
